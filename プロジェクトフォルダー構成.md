## 📋 フォルダー説明

### 🌱 smart-planter/ (メインプロジェクト)
- プロジェクトのルートディレクトリ
- 設定ファイル、メイン実行ファイルを配置

### 📁 src/ (ソースコード)
- **app/**: Flaskアプリケーションのメインコード
- **sensors/**: センサー制御関連のコード
- **watering/**: 給水制御関連のコード
- **camera/**: カメラ制御関連のコード
- **notifications/**: 通知機能関連のコード
- **data/**: データ管理関連のコード
- **api/**: REST API関連のコード
- **utils/**: 共通ユーティリティ関数
- **web/**: Webフロントエンド関連

### 📁 tests/ (テストコード)
- 各機能の単体テスト
- 統合テスト
- 長期稼働テスト

### 📁 docs/ (ドキュメント)
- API仕様書
- デプロイメントガイド
- トラブルシューティング
- ユーザーマニュアル

### 📁 scripts/ (スクリプト)
- セットアップスクリプト
- 起動・停止スクリプト
- バックアップ・クリーンアップスクリプト

### 📁 logs/ (ログファイル)
- アプリケーションログ
- エラーログ
- アクセスログ

### 📁 data/ (データファイル)
- 開発・テスト用のデータファイル
- 本番環境ではUSBストレージを使用

### 📁 config/ (設定ファイル)
- 環境別設定ファイル
- 開発・本番・テスト環境の設定

## 🎯 設計の特徴

### 1. **機能分離**
- 各機能を独立したモジュールとして分離
- チームメンバーが並行して作業可能

### 2. **標準準拠**
- Pythonプロジェクトの標準的な構成
- Flaskアプリケーションの推奨構成

### 3. **拡張性**
- 新機能の追加が容易
- 設定の変更が簡単

### 4. **保守性**
- コードの場所が明確
- テストコードが整理されている

### 5. **デプロイ対応**
- 本番環境への展開が容易
- 環境別設定の管理

## 📝 命名規則

### ファイル名
- **Pythonファイル**: snake_case (例: sensor_manager.py)
- **HTMLファイル**: snake_case (例: dashboard.html)
- **CSS/JSファイル**: snake_case (例: style.css)
- **設定ファイル**: snake_case (例: config.py)

### ディレクトリ名
- **小文字**: すべて小文字で記述
- **アンダースコア**: 複数単語はアンダースコアで区切り
- **短縮**: 必要に応じて適切に短縮

## 🔧 使用方法

### 開発環境での使用
```bash
cd smart-planter
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

### テスト実行
```bash
cd smart-planter
python -m pytest tests/
```

### 本番環境での使用
```bash
cd smart-planter
sudo systemctl start smart-planter
```
